Dominios Prevy (un solo sitio Netlify, ver netlify.toml)
=========================================================

Plataforma (PWA / app)
----------------------
- Host: https://plataforma.prevy.cl
- Netlify → Domain management → Add domain alias → plataforma.prevy.cl
- DNS zona prevy.cl: CNAME `plataforma` → tu-sitio.netlify.app (o lo que indique Netlify)
- SSL en verde → probar https://plataforma.prevy.cl/

Landing marketing Prevy
-----------------------
- Hosts: https://www.prevy.cl y https://prevy.cl (apex)
- Añadir ambos en Netlify; DNS: CNAME www → netlify; apex según proveedor (ALIAS/ANAME o A de Netlify)

Landing Hablemos depensión (sin cambios de contenido)
-----------------------------------------------------
- Hosts: https://www.hablemosdepension.cl y https://hablemosdepension.cl usan la misma landing /www/index.html que antes (catch-all).
- El enlace «Acceso equipo» del pie usa PREVY_PLATAFORMA_ORIGIN → https://plataforma.prevy.cl (www/site-urls.js).

Legacy (opcional mientras migres DNS)
-------------------------------------
- plataforma.hablemosdepension.cl / plataforma.hablemospensiones.cl siguen en netlify.toml → misma app.

www/site-urls.js
----------------
PREVY_PLATAFORMA_ORIGIN = https://plataforma.prevy.cl
(La landing en /www/index.html usa esto para data-app-href «Acceso equipo» en previews.)

Supabase → Authentication → URL Configuration
---------------------------------------------
- Site URL recomendada: https://plataforma.prevy.cl
- Redirect URLs: https://plataforma.prevy.cl/** y mantén las URLs antiguas hasta completar la migración.

Mercado Pago / webhooks / Telegram
----------------------------------
- Actualizar URLs públicas a https://plataforma.prevy.cl donde corresponda.

Web Push (recordatorios PWA)
----------------------------
- Supabase SQL: supabase-migration-push-subscriptions.sql
- Netlify → Environment: VAPID_PUBLIC_KEY, VAPID_PRIVATE_KEY, VAPID_SUBJECT (mailto:… o https://…)
- Generar claves: npx web-push generate-vapid-keys
- supabase-config.js: window.PREVY_VAPID_PUBLIC_KEY = (misma clave pública que en Netlify)
- Tras login, el dashboard pide permiso la primera vez por usuario/navegador y guarda la suscripción vía save-push-subscription.
